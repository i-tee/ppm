# Личный кабинет партнёра для управления партнёрской программой

> **Операционная документация** (окружения, ручной прод-деплой, интеграция
> с основным бэкендом Avicenna, тест-чеклист) — **[docs/operations.md](docs/operations.md)**.
> Стратегия миграции партнёрки (Фаза D) — `avicenna-backend/docs/partner-migration-plan.md`.

## Обзор
Это реактивное одностраничное приложение (SPA) на Laravel 12 и Vue.js, предназначенное для управления партнёрскими программами. Личный кабинет создан для специалистов, рекламщиков, блогеров и инфлюенсеров, которые зарабатывают на реализации брендовых товаров. Приложение предоставляет инструменты для работы с промокодами, начислениями бонусов и партнёрскими ссылками. Оно гибко настраивается и может быть адаптировано под любой бизнес или торговый бренд.

## Технологический стек
- **Бэкенд**: 
  - Laravel 12 с Sanctum для токен-базированной авторизации.
  - PHP 8.2 или выше.
  - MySQL (или любой другой поддерживаемый Laravel движок базы данных).
- **Фронтенд**: 
  - Vue.js 3 для создания реактивного интерфейса.
  - Vite 6.x для сборки и разработки.
  - Pinia для управления состоянием.
  - Axios для работы с API.
  - Vuestic UI для стилизации компонентов и адаптивного дизайна.
  - i18n для поддержки локализации интерфейса.
- **Дополнительно**: 
  - Node.js 20.x (см. `.nvmrc`).
  - NPM для управления зависимостями JavaScript.

## Текущие возможности (боевые, прод — partner.avicenna.com.ru)
- **Авторизация**: регистрация/вход по токенам (Sanctum) + соц-вход (Yandex; Google отключён), верификация email, восстановление пароля.
- **Промокоды**: партнёр сам создаёт процентные и бонусные промокоды из ЛК. С Фазы D (2026-07-16) — **dual-write**: минт через API основного бэкенда Avicenna (источник истины) + дублирование в БД Joomla — код работает и на старом, и на новом сайте. Журнал минтов `minted_coupons` (идемпотентность по `mint_request_id`).
- **Начисления и баланс**: живой пересчёт из двух источников — история из БД Joomla (заказы старого сайта) + начисления/сторно/погашения из API нового бэкенда (`/partner/accruals`, `/partner/redemptions`). Вкладки «Начисления», «Списания» (с «Корректировками»), статусы бонусников с учётом гашения на новом сайте.
- **Выплаты**: заявки на вывод с воркфлоу статусов, комиссия по типу лица (физлицо/самозанятый/ИП/ООО), чеки самозанятых, реквизиты.
- **Админ-зона**: обработка заявок партнёров и выплат, импersonate, роль-гейт `EnsureUserIsAdmin` на `/api/admin/*`.
- **Локализация**: ru/en через i18n.

## Планы развития
- **Партнёрские ссылки**: генерация реферальных ссылок для отслеживания трафика и продаж.
- **Дашборд-аналитика**: расширенная панель (доходы, клики, конверсии).
- **Партнёрка 2.0**: боты TG/MAX, витрины (см. `avicenna-backend/docs/roadmap-after-launch.md`).

## Требования
### Бэкенд
- **Laravel**: Версия 12.x с установленным Sanctum.
- **PHP**: 8.2 или выше.
- **MySQL**: Для хранения данных.
- **Composer**: Для управления зависимостями PHP.
- **Nginx/Apache**: Настроенный с поддержкой SSL и CORS.

### Фронтенд
- **Node.js**: Версия 20.x (см. `.nvmrc`).
- **NPM**: Для управления зависимостями JavaScript.
- **Vite**: Версия 6.x.

### Окружение
- **Домен**: Настроенный с поддержкой HTTPS.
- **API-эндпоинт**: Доступный по пути `/api` с правильной конфигурацией CORS.
